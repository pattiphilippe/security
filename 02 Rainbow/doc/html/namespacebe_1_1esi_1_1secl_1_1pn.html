Attempts to find passwords corresponding to the hashes given, at the hand of the tails.

Using a opened sqlite rainbow table, should have been created with generateRT to have a good structure. Crack is performed in multiple threads.

Parameters
hashFileThe file's name containing the hashes to crack.
dbThe db containing the rainbow table.
crackedPwdFileThe file's name for the cracked passwords.
crackedHashFileThe file's name for the hashes of the cracked passwords. Each line is the hash of the password of the same line of the passwords file.
pwdSizeThe passwords's lenght of the table.
nbReduceThe number of reduce to apply before getting the tail.
Exceptions
std::runtime_errorif hashFile, headFile, tailsFile or crackedFile can't be opened.

)

Generate the head and the tails of the RT, and write them into the DB.

The tails are computed after a number of reductions, based on their hash. Drop the table if exists. When a merge occure, the row is not put into the DB.

Parameters
dbThe db to store the passwords and the tails. It must be a valid db.
pwdSizeThe passwords's lenght of the table.
nbHeadThe number of tries to generate a line, so the final number of lines can be lower, due to same heads or tails. It should be a multiple of NB_THREADS_GENERATE.
nbReduceThe number of reduction functions to apply to compute the tail. If not set, use default value.
